# API docs DRAFT

## ВОПРОСЫ:
 - будет ли реализована авторизация пользователей?

## Endpoints & methods

### Users

- GET /users – получить всех пользователей (только список)
- GET /users/id – получить одного пользователя (полная информация о пользователе)
- GET /users/me – информация о текущем пользователе (если будет имплементация авторизации)

- POST /users – создать пользователя
- DELETE /users/id – удалить пользователя
- PATCH /users/id – изменить данные пользователя (username)

- PATCH /users/id/update_password – изменить пароль пользователя
или
- PATCH /users/update_password – изменить пароль текущего пользователя (если будет имплементация авторизации)

- POST /users/id/courses – добавить курс для пользователя
или 
- POST /users/courses – добавить курс для текущего пользователя (если будет имплементация авторизации)

- DELETE /users/user_id/courses/course_id - удалить курс для пользователя
или
- DELETE /users/courses/course_id - удалить курс для текущего пользователя (если будет имплементация авторизации)


#### получить список тренировок со статусом для конкретного пользователя и курса
Варианты:
- GET /users/user_id/courses/course_id/workouts - получить список тренировок со статусом для конкретного пользователя и курса
или
- GET /workouts/?user=1&course=1 – получить список тренировок со статусом для конкретного пользователя и курса
или
- GET /courses/id/workouts – получить список тренировок со статусом для конкретного курса и текущего пользователя (если будет авторизация)

- GET /courses/course_id/workouts/workout_id – информация о тренировке по конкретному курсу для текущего пользователя (если будет авторизация)

- PATCH /courses/course_id/workouts/workout_id – обновить прогресс по тренировке по конкретному курсу для текущего пользователя (если будет авторизация)


### Courses - курсы

- GET /courses – получить все курсы (только список)
- GET /courses/id - получить один курс (полная информация о курсе)
